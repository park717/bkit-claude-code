## 1. User Intent Discovery

### 1.1 핵심 문제

bkit v1.6.2는 강력한 PDCA 인프라를 갖추고 있으나, 세 가지 근본적 Gap이 존재:

1. **자동화 Gap**: 모든 PDCA 단계 전환에 사용자의 명시적 명령이 필요. `shouldAutoAdvance()`가 존재하지만 실제 자동 전환은 check→act만 가능
2. **투명성 Gap**: AI가 무엇을 하고 있는지, 왜 그 결정을 했는지 알 수 없음. Agent Team 내부가 블랙박스
3. **시각화 Gap**: PDCA 진행 상황이 텍스트 기반으로만 표시. `/pdca status`를 호출해야 확인 가능

### 1.2 타겟 사용자

| 사용자 유형 | 사용 컨텍스트 | 핵심 니즈 | 우선순위 |
|------------|-------------|----------|:--------:|
| 중급 풀스택 개발자 | 스타트업, Claude Code + Cursor 혼용 | 속도와 품질 동시 달성, 문서-코드 동기화 | 1 |
| 1인 창업자/인디해커 | MVP 빠른 출시, 혼자 전체 담당 | "가상 CTO 팀" 경험, 반복 가능한 개발 시스템 | 2 |
| 테크리드/CTO | 5-15인 팀, 개발 표준화 책임 | 거버넌스, 감사 추적, 팀 품질 일관성 | 3 |
| 초보 개발자 | 부트캠프 수료, AI 코딩 입문 | 체계적 프로세스 강제화, 품질 피드백 | 4 |
| 비개발자 PM/디자이너 | 프로토타입 직접 제작 원함 | 자연어→구조화된 개발 계획 변환 | 5 (v3.0) |

## 2. Alternatives Explored

### 2.1 Approach A: 상태 머신만 구현 (DSL 없음)

| 측면 | 상세 |
|------|------|
| **요약** | 순수 JavaScript 기반 상태 머신만 도입, 워크플로우 커스터마이징 없음 |
| **장점** | 구현 단순 (5-7일), 기존 코드 최소 변경, 테스트 용이 |
| **단점** | 워크플로우 커스터마이징 불가, 핫픽스/Enterprise 구분 하드코딩, 장기 확장성 제한 |
| **복잡도** | Low |
| **적합 상황** | 빠른 출시가 최우선, 커스텀 워크플로우 불필요 시 |

### 2.2 Approach B: 상태 머신 + 경량 DSL — 선택

| 측면 | 상세 |
|------|------|
| **요약** | 상태 머신 + YAML 기반 워크플로우 정의 언어 (커스텀 워크플로우 지원) |
| **장점** | 커스텀 워크플로우 지원 (default, hotfix, enterprise), YAML 기반 접근성, Enterprise 차별화 |
| **단점** | 구현 복잡도 증가 (12-17일), DSL 파서 유지보수 필요, 초기 학습 곡선 |
| **복잡도** | Medium |
| **적합 상황** | Enterprise 고객 타겟, 다양한 워크플로우 필요 시 |

### 2.3 Approach C: 외부 워크플로우 엔진

| 측면 | 상세 |
|------|------|
| **요약** | 기존 검증된 워크플로우 엔진 (xstate, temporal 등) 활용 |
| **장점** | 검증된 엔진 사용, 풍부한 기능 |
| **단점** | 외부 의존성 추가 (bkit 철학 위반), CC 플러그인 환경 제약 |
| **복잡도** | High |
| **적합 상황** | 복잡한 분산 워크플로우 필요 시 |

### 2.4 결정 근거

**선택**: Approach B (상태 머신 + 경량 DSL)

**이유**: bkit의 "Automation First" 철학에 부합하며, CC 플러그인 내에서 자체 완결적 솔루션 제공. Enterprise 사용자의 커스텀 워크플로우 요구에 대응하면서도 npm 외부 의존성 없이 구현 가능.

## 12. "통제 가능한 AI" 4원칙

v2.0.0의 모든 자동화 기능은 아래 4원칙을 준수해야 합니다:

| 원칙 | 설명 | 구현 |
|------|------|------|
| **안전 기본값** | 기본은 항상 더 안전한 방향 (L2 Semi-Auto) | 권한 확대는 명시적 승인, 축소는 자동 |
| **점진적 신뢰** | 신뢰는 얻는 것이지 주어지는 것이 아님 | Trust Score 기반, 성공 쌓여야 레벨업 |
| **완전한 가시성** | 모든 AI 행동은 추적 가능해야 함 | Decision Trace, 감사 로그, Plan Preview |
| **언제나 중단 가능** | 어떤 레벨에서도 즉시 중단+롤백 가능 | 체크포인트, Emergency Stop, /pdca rollback |
